CAUSAS POSIBLES:

1. CACHE DE BASH (PROBABLE)
   - Terminal no actualizó lista de carpetas
   - Solución: hash -r

2. PERMISOS DE CARPETA (POSIBLE)
   - Carpeta existe pero usuario no tiene permisos de lectura
   - Solución: chmod -R 755 ~/public_html/laravel_app

3. CLONING INCOMPLETO (POCO PROBABLE)
   - Clonó pero falló antes de terminar
   - Solución: rm -rf ~/public_html/laravel_app && git clone...

4. CARPETA EN OTRO LUGAR (RARO)
   - Está en /home en lugar de /home/u519347385/public_html
   - Solución: find ~ -name "laravel_app"

5. DOCUMENTO ROOT DIFERENTE (POSIBLE)
   - Hostinger tiene document root en otro sitio
   - Solución: Revisar configuración de Hostinger

PASO 3: BASED ON DIAGNOSTICO...

Si EXISTE en ~/public_html/laravel_app:
  - Arreglar permisos
  - chmod -R 755 ~/public_html/laravel_app
  - Intentar cd nuevamente

Si EXISTE en otro lugar:
  - Mover a public_html
  - mv /ruta/actual ~/public_html/laravel_app

Si NO EXISTE en ningún lado:
  - Clonar desde GitHub
  - cd ~/public_html
  - git clone https://github.com/guilleq18/mobels_alejandro.git laravel_app


PASO 4: VERIFICAR QUE ESTÁ EN LUGAR CORRECTO

Debe mostrar carpeta "app" (código de Laravel)
